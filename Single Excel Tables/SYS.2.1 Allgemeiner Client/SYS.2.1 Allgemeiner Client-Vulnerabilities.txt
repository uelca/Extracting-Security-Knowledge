Schwachstelle für Maßnahme: SYS.2.1.A1 und Bedrohung: G 0.30
Schwachstellen:

1. Unzureichende Passwortrichtlinien: Wenn es keine klaren Vorgaben für die Erstellung starker Passwörter gibt und Benutzer schwache oder leicht zu erratende Passwörter verwenden dürfen, besteht die Schwachstelle darin, dass ein unbefugter Benutzer leicht Zugriff auf das System erlangen kann.

2. Mangelnde regelmäßige Überprüfung von Berechtigungen: Wenn die Berechtigungen von Benutzern nicht regelmäßig überprüft und aktualisiert werden, können sich Berechtigungen ansammeln, die nicht mehr benötigt werden. Dadurch erhöht sich das Risiko einer unberechtigten Nutzung des Systems durch Personen, denen eigentlich der Zugriff verwehrt sein sollte.

3. Schwache Zutrittskontrolle: Wenn physische Zugänge zu Geräten und Systemen nicht ausreichend geschützt sind, kann dies zu einer unberechtigten Nutzung oder Administration führen. Beispielsweise könnten unbefugte Personen unbeobachtet Zugriff auf Systeme oder Geräte erlangen und diese nutzen.

4. Unzureichende Schulung der Benutzer: Wenn Benutzer nicht ausreichend darüber aufgeklärt werden, wie sie sich sicher anmelden und ihre Zugangsdaten schützen können, besteht die Schwachstelle darin, dass sie leicht Opfer von Phishing-Angriffen oder Social Engineering werden und ihre Zugangsdaten an unbefugte Dritte weitergeben könnten.

5. Unsichere Aufbewahrung von Zugangsdaten: Wenn Benutzer ihre Zugangsdaten unsicher aufbewahren, beispielsweise auf einem Zettel neben dem Computer, besteht die Gefahr, dass diese Informationen in die falschen Hände gelangen und unberechtigte Benutzer Zugriff auf das System erhalten.

Diese Schwachstellen können durch die Maßnahme der sicheren Benutzerauthentisierung angegangen werden, indem geeignete Passwortrichtlinien und Passwortmanagement-Tools implementiert werden, regelmäßige Überprüfungen von Berechtigungen durchgeführt werden, physische Zugänge zu Geräten und Systemen geschützt werden, die Schulung der Benutzer verbessert wird und Möglichkeiten zur sicheren Aufbewahrung von Zugangsdaten bereitgestellt werden.


Schwachstelle für Maßnahme: SYS.2.1.A3 und Bedrohung: G 0.28
1. Schwachstelle: Mangelnde Kontrolle der Autoupdate-Mechanismen.
Beschreibung: Es besteht die Möglichkeit, dass die Autoupdate-Mechanismen nicht angemessen kontrolliert werden und schädlicher Code in Form von Updates eingeschleust wird. Dadurch können Angreifer die Software manipulieren oder Zugriff auf das System erlangen.

2. Schwachstelle: Fehlende Überprüfung der Update-Quellen.
Beschreibung: Wenn die Software nicht überprüft, ob die Updates von vertrauenswürdigen Quellen stammen, besteht die Gefahr, dass bösartiger Code eingeschleust wird. Angreifer können hierbei gefälschte Updates bereitstellen und Schadsoftware auf dem System installieren.

3. Schwachstelle: Fehlende Versionskontrolle der installierten Updates.
Beschreibung: Ohne eine effektive Versionskontrolle kann es zu Fehlern bei der Installation von Updates kommen. Angreifer könnten diese Schwachstelle ausnutzen, um ältere, unsichere Versionen der Software einzuschleusen oder Updates zu blockieren, die Sicherheitslücken beheben.

4. Schwachstelle: Fehlende Möglichkeit zur Deaktivierung der Autoupdate-Mechanismen.
Beschreibung: Wenn Benutzer keine Kontrolle über die Aktivierung oder Deaktivierung der Autoupdate-Mechanismen haben, können Angreifer diese Schwachstelle ausnutzen, um schädlichen Code einzuschleusen. Benutzer sollten die Möglichkeit haben, Autoupdate-Funktionen nach Bedarf zu aktivieren oder deaktivieren.

5. Schwachstelle: Unzureichendes Logging der Aktivitäten der Autoupdate-Mechanismen.
Beschreibung: Ohne ein angemessenes Logging kann es schwer sein, Anomalien oder verdächtige Aktivitäten im Zusammenhang mit den Autoupdate-Mechanismen zu erkennen. Dadurch könnten Angriffe unentdeckt bleiben und Schadcode auf dem System installiert werden.


Schwachstelle für Maßnahme: SYS.2.1.A6 und Bedrohung: G 0.39
1. Schwachstelle: Unaktualisierte Schutzprogramme
Beschreibung: Wenn die vorhandenen Schutzprogramme auf dem Client nicht regelmäßig aktualisiert werden, können sie nicht effektiv vor neuen Schadprogrammen schützen. Eine veraltete Version eines Schutzprogramms kann Schwachstellen aufweisen, die von Angreifern ausgenutzt werden können.

2. Schwachstelle: Fehlende Überwachung und Alarmierung
Beschreibung: Wenn das eingesetzte Schutzprogramm keine ausreichende Überwachung und Alarmierung bietet, kann nicht rechtzeitig auf eine Bedrohung reagiert werden. Ein Mangel an Alarmen über schädliche Aktivitäten ermöglicht es Angreifern, unerwünschte Funktionen auszuführen, ohne dass dies bemerkt wird.

3. Schwachstelle: Unzureichender Schutz vor Zero-Day-Angriffen
Beschreibung: Schutzprogramme gegen Schadsoftware können möglicherweise nicht vor Zero-Day-Angriffen schützen. Zero-Day-Angriffe nutzen neu entdeckte Schwachstellen aus, für die bisher keine Sicherheitsupdates oder Definitionen verfügbar sind. Wenn das eingesetzte Schutzprogramm nicht in der Lage ist, solche Angriffe zu erkennen und zu blockieren, bleibt das System verwundbar.

4. Schwachstelle: Fehlkonfiguration der Schutzprogramme
Beschreibung: Eine fehlerhafte Konfiguration der eingesetzten Schutzprogramme kann zu Schwachstellen führen, die von Angreifern ausgenutzt werden können. Beispielsweise kann eine falsche Einstellung dazu führen, dass das Schutzprogramm bestimmte Arten von Schadprogrammen nicht erkennt oder blockiert.

5. Schwachstelle: Mangelnder Benutzerschulung und sensibilisierung
Beschreibung: Wenn die Benutzerinnen und Benutzer nicht ausreichend über die Gefahren von Schadprogrammen und die ordnungsgemäße Nutzung der Schutzprogramme informiert sind, können sie versehentlich Maßnahmen ergreifen, die die Wirksamkeit der Schutzmaßnahmen beeinträchtigen. Eine unzureichende Schulung und Sensibilisierung erhöht das Risiko einer erfolgreichen Infektion durch Schadprogramme.


Schwachstelle für Maßnahme: SYS.2.1.A8 und Bedrohung: G 0.22
Schwachstellen für die Gefährdung "Manipulation von Informationen":

1. Unzureichende Zugriffskontrolle: Personen haben unbefugten Zugriff auf Informationen, auf die sie keine Berechtigung haben. Dadurch können sie Informationen manipulieren, ohne dass es frühzeitig erkannt wird.

2. Schwache Authentifizierung: Das IT-System erlaubt schwache oder leicht zu knackende Passwörter, was Angreifenden ermöglicht, sich als autorisierte Benutzer auszugeben und Informationen zu manipulieren.

3. Fehlende Überwachung und Protokollierung: Es gibt keine angemessene Überwachung und Protokollierung von Änderungen an Informationen, daher können Manipulationen unbemerkt bleiben.

4. Fehlerhafte Datenvalidierung: Das IT-System überprüft nicht ausreichend die Gültigkeit und Integrität der eingegebenen Daten, wodurch manipulierte Informationen eingeführt werden können.

5. Unzureichende Sicherheit der Speichermedien: Externe Speichermedien oder Laufwerke können von beliebigen Personen verwendet werden, ohne dass dies kontrolliert wird. Dadurch können Angreifende manipulierte Daten einschleusen.

Schwachstellen, die durch die Maßnahme "Absicherung des Bootvorgangs" gemindert werden:

1. Unerlaubtes Booten von nicht autorisierten Medien: Durch die Festlegung, von welchen Medien gebootet werden darf und die Beschränkung auf Administratoren, wird verhindert, dass Personen mit unautorisierten Medien das System starten und Informationen manipulieren können.

2. Unbefugte Änderungen der Boot-Konfiguration: Die Einschränkung der Konfigurationsänderungen auf Administratoren stellt sicher, dass keine unbefugten Personen den Bootvorgang manipulieren können.

3. Deaktivierte nicht benötigte Funktionen in der Firmware: Durch die Deaktivierung unnötiger Funktionen wird die Angriffsfläche reduziert und mögliche Manipulationen des Bootvorgangs erschwert.

Schwachstelle für Maßnahme: SYS.2.1.A10 und Bedrohung: G 0.18
Schwachstellen und ihre Erläuterungen:

1. Schwachstelle: Fehlende Dokumentation der Entscheidungen
Erklärung: Wenn die Entscheidungen, die in der Planungsphase getroffen wurden, nicht angemessen dokumentiert werden, besteht die Gefahr, dass sie später nicht nachvollzogen werden können. Dies kann zu Unsicherheiten bei der Umsetzung führen und ermöglicht es potenziellen Angreifern, die Planung zu manipulieren.

2. Schwachstelle: Vernachlässigung betrieblicher Aspekte
Erklärung: Wenn bei der Planung des Einsatzes von Clients nur IT- oder Informationssicherheitsaspekte berücksichtigt werden und betriebliche Anforderungen außer Acht gelassen werden, besteht die Gefahr einer Fehlkonfiguration oder unzureichenden Einbindung in die bestehenden Prozesse. Dadurch können Sicherheitsprobleme entstehen, die von Angreifern ausgenutzt werden können.

3. Schwachstelle: Mangelhafte Definition der organisatorischen Abläufe
Erklärung: Wenn die organisatorischen Abläufe, die der Informationsverarbeitung dienen, fehlerhaft definiert sind, können Sicherheitsprobleme auftreten. Dies bedeutet, dass selbst wenn jeder einzelne Prozessschritt korrekt durchgeführt wird, das Gesamtsystem anfällig für Angriffe ist. Eine klare und umfassende Definition der Abläufe ist daher erforderlich, um Schwachstellen zu mindern.


Schwachstelle für Maßnahme: SYS.2.1.A11 und Bedrohung: G 0.18
1. Schwachstelle: Unvollständige oder fehlerhafte Erstellung der Anforderungsliste
Erläuterung: Eine unvollständige oder fehlerhafte Erstellung der Anforderungsliste kann dazu führen, dass wichtige Kriterien für die Bewertung der am Markt erhältlichen Produkte nicht berücksichtigt werden. Erläuterung: Wenn die herstellenden Unternehmen von IT- und Betriebssysteme nicht zeitnah Patches für Schwachstellen bereitstellen, können bekannte Sicherheitslücken nicht geschlossen werden. Dadurch sind die beschafften Clients anfällig für Angriffe, die auf bekannte Schwachstellen abzielen.

Erläuterung: Die Verwendung von Betriebssystemen mit einem Rolling-Release-Modell bedeutet, dass die Systeme kontinuierlich aktualisiert werden, ohne dass klare Kontrollpunkte für die Evaluierung von Patches existieren. 4. Schwachstelle: Fehlende oder unzureichende Firmware-Konfigurationsoberfläche
Erläuterung: Wenn die zu beschaffenden Systeme keine oder nur eine unzureichende Firmware-Konfigurationsoberfläche für UEFI SecureBoot und das TPM haben, kann die Institution keine wirksame Kontrolle über diese Sicherheitsmechanismen ausüben. Dadurch können potenzielle Sicherheitslücken nicht effektiv gemindert oder genutzt werden.

2. Schwachstelle: Fehlende Aktualisierung der Verschlüsselung
Erläuterung: Wenn die verwendete Verschlüsselungstechnologie nicht regelmäßig aktualisiert wird, könnten Sicherheitslücken auftreten, die von Angreifern ausgenutzt werden können.

3. Schwachstelle: Phishing-Angriffe auf verschlüsselte Kommunikation
Erläuterung: Wenn Angreifer erfolgreich Phishing-Angriffe durchführen und Benutzer dazu bringen, ihre Anmeldeinformationen preiszugeben, können sie Zugang zu verschlüsselten Kommunikationsverbindungen erhalten.

4. Schwachstelle: Malware auf Endgeräten
Erläuterung: Wenn Endgeräte mit Malware infiziert sind, können Angreifer die verschlüsselte Kommunikation abfangen, bevor sie verschlüsselt oder nachdem sie entschlüsselt wurde.

5. Schwachstelle: Unsichere Netzwerkinfrastruktur
Erläuterung: Wenn das Netzwerk, über das die Kommunikation erfolgt, unsicher ist (z.B. ungesicherte WLAN-Verbindungen), können Angreifer die Kommunikation abfangen, bevor sie verschlüsselt wurde.

10. Mangelnde Verschlüsselung von administrativen Verbindungen: Fehlende Verschlüsselung von administrativen Verbindungen kann es Angreifern ermöglichen, den Datenverkehr abzufangen und die Administrationsverfahren zu manipulieren.
